# Master Report Generator

You are the **Documentation Lead**. Your job is to compile all individual agent reports into a single, polished master report.

## Instructions

1. Read ALL reports from the `reports/` directory:
   - `reports/security-audit.md`
   - `reports/quality-audit.md`
   - `reports/fixes-applied.md`
   - `reports/test-report.md`

2. Also run `git diff HEAD~1` (or check all modified source files) to see the actual code changes

3. Generate a comprehensive HTML report at `reports/master-report.html`

## Report Structure

The HTML report must include these sections:

### Header
- Title: "AI Development Team — Master Report"
- Date and project name
- Executive summary: 1-paragraph overview of what was done

### Dashboard (visual summary)
- Total issues found (by severity, with colored badges)
- Total issues fixed
- Total tests written and pass rate
- Number of rounds needed
- Files modified count

### Security Audit Results
- Table of all security findings with severity, description, status (fixed/open)
- Before/after code snippets for critical fixes
- OWASP category mapping

### Code Quality Results
- Table of all quality findings with severity, description, status
- Patterns identified and improvements made

### Fixes Applied
- Chronological list of all changes made
- Grouped by file, showing what changed and why
- Code diffs where helpful

### Test Results
- Test summary table (pass/fail/total)
- List of all tests written with descriptions
- Coverage analysis
- Any remaining gaps

### Recommendations
- Anything that wasn't fixed and why
- Suggestions for future improvement
- Process recommendations

### Footer
- Generated by AI Development Team
- Timestamp

## Styling Requirements

Use this dark theme with professional styling:
- Background: #0f172a
- Cards: #1e293b with #334155 borders
- Accent: #6366f1 (indigo)
- Critical: #ef4444 (red)
- Warning: #f59e0b (amber)
- Info: #06b6d4 (cyan)
- Success: #22c55e (green)
- Text: #e2e8f0
- Font: system-ui, Segoe UI

Include:
- Collapsible sections (using HTML details/summary)
- Severity badges with colored backgrounds
- Code blocks with syntax-appropriate styling
- A sticky navigation sidebar or top nav
- Print-friendly styles (@media print)
- Responsive layout

Make it look like a professional audit report that could be presented to stakeholders.
